Hera (the login/consent UI) has 10 page routes. Each renders Kratos self-service flows or handles Hydra challenges.

## Routes

| Path | Description |
|------|-------------|
| [`/settings/security`](./settings-security) | - |
| [`/settings`](./settings) | User settings, password, MFA, profile, social IdP linking. |
| [`/registration`](./registration) | Renders the Kratos registration flow. |
| [`/recovery`](./recovery) | Renders the Kratos recovery (password-reset) flow. |
| [`/logout`](./logout) | Renders or auto-confirms the Hydra logout challenge. |
| [`/verification`](./verification) | Renders the Kratos verification (email-confirm) flow. |
| [`/consent`](./consent) | Renders or auto-grants the Hydra OAuth2 consent challenge. |
| [`/reset-password`](./reset-password) | - |
| [`/login`](./login) | Renders the Kratos login flow + Hydra login challenge handler. |
| [`/link-conflict`](./link-conflict) | - |
