Athena's `src/features/` directory holds per-feature code. Each module is a self-contained Next.js page + components + actions + tests.

## Modules

| Module | Description |
|--------|-------------|
| [`analytics`](./analytics) | Dashboard analytics widgets — PKCE traffic, MFA stats, login attempts, lockout t |
| [`auth`](./auth) | Athena's own auth chain. Login via OAuth2 against IAM Hera; session cookie manag |
| [`identities`](./identities) | Identity CRUD UI — list, view, edit, delete identities in either CIAM or IAM Kra |
| [`m2m-clients`](./m2m-clients) | Machine-to-machine OAuth2 client management. Create, view, rotate secrets. |
| [`messages`](./messages) | Kratos courier message log — verification emails, recovery emails, notification  |
| [`oauth2-auth`](./oauth2-auth) | Background module handling the OAuth2 flow into Athena itself. |
| [`oauth2-clients`](./oauth2-clients) | User-facing OAuth2 client management UI. |
| [`oauth2-tokens`](./oauth2-tokens) | OAuth2 token introspection — paste a token, see its claims. |
| [`schemas`](./schemas) | Identity schema editor with live reload via the Kratos reload sidecar. |
| [`security`](./security) | Security audit log viewer. |
| [`sessions`](./sessions) | Active sessions list with revoke action. |
| [`settings`](./settings) | Settings vault editor for the Olympus settings DB. |

See [Internals — Athena features](/docs/internals/athena-features) for the architectural pattern.
