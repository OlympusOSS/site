# `session_aal2_required`

**Source:** Kratos

**HTTP:** 403

## Summary

Session AAL is too low for the requested operation.

## When this fires

AAL1 session trying to perform a settings change that requires AAL2.

## Fix

Step up via `/self-service/login/browser?aal=aal2&refresh=true`. See [Troubleshooting, AAL too low](/docs/troubleshooting/auth-issues/kratos-session-aal-too-low).
