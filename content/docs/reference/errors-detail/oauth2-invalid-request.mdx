# `invalid_request`

**Source:** Hydra (RFC 6749)

**HTTP:** 400

## Summary

OAuth2 request is missing or has malformed parameters.

## When this fires

Common with bad authorize/token requests.

## Common causes

- Missing client_id
- Missing redirect_uri
- Missing PKCE challenge for public client

## Fix

Check the request parameters. See [Troubleshooting, pkce_required](/docs/troubleshooting/oauth2-issues/oauth2-pkce-required) for the PKCE case.
