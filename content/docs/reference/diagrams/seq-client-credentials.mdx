---
title: Client Credentials (M2M)
description: Server-to-server OAuth2 flow
---

```mermaid
sequenceDiagram
    autonumber
    participant Worker as Backend worker
    participant Hydra

    Note over Worker: Time to call API X
    Worker->>Worker: Check cached token
    alt Token cached + valid
        Note over Worker: Use cached access_token
    else No cache or expired
        Worker->>Hydra: POST /oauth2/token<br/>grant_type=client_credentials<br/>Basic auth: client_id:client_secret
        Hydra->>Hydra: Verify client credentials
        Hydra->>Hydra: Check allowed scopes
        Hydra-->>Worker: { access_token, expires_in: 3600, scope: "..." }
        Worker->>Worker: Cache for ~55 min
    end

    Worker->>API: GET /resource<br/>Authorization: Bearer <token>
    API->>Hydra: POST /oauth2/introspect (or local JWT verify)
    Hydra-->>API: { active: true, sub: client_id, scope: "..." }
    API-->>Worker: 200 + data
```

## Properties

- **No user involved.** The `sub` claim is the `client_id`, not a user UUID.
- **No refresh token.** Worker just requests a new one when needed.
- **No ID token.** Pure machine identity.
- **Short TTL** (1 hour default). Cache and renew on expiry.
