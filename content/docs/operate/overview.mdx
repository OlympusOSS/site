Operating an identity platform is a different shape of work from building one. This section is the runbook collection — the things you do on a Tuesday afternoon when something breaks, when a key needs to rotate, when an account is locked.

## When you are paged

- [Incident Response](/docs/operate/incident-response) — the on-call playbook.
- [Health Endpoints](/docs/operate/health-endpoints) — what each service's `/health` does and doesn't tell you.
- [Network Topology](/docs/operate/network-topology) — which ports must be firewalled, and which exist for in-cluster traffic only.

## Scheduled rotations

- [Certificate rotation](/docs/operate/cert-rotation) — Caddy + database TLS certs.
- [Session signing key rotation](/docs/operate/session-signing-key-rotation) — zero-downtime.
- [Encryption key rotation](/docs/operate/encryption-key-rotation) — for the SDK settings vault.
- [Reload API key rotation](/docs/operate/reload-api-key-rotation) — for the Kratos schema reload sidecar.
- [Secrets audit](/docs/operate/secrets-audit) — quarterly inventory.

## User support

- [Locked account unlock](/docs/operate/locked-account-unlock) — diagnose and unlock a customer or employee.
- [pgAdmin DBA offboarding](/docs/operate/pgadmin-dba-offboarding) — remove a departing DBA's database access cleanly.

- [Logs and observability](/docs/operate/logs-and-observability) — what's logged, where it goes, retention.
